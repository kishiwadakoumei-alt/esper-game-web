# ブラウザフロントエンド

第5段階では、FastAPIが返す公開ゲーム状態を使用するブラウザ画面を追加した。
現在はFastAPIを正式なアプリケーション入口とし、
通常のHTML/CSS/JavaScriptを配信する。

## 起動

```bash
uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

ブラウザで `http://localhost:8000/` を開く。
APIのヘルスチェックは引き続き `GET /api/health`、
OpenAPIは `http://localhost:8000/docs` で確認できる。

## ファイル構成

```text
frontend/
├── index.html
└── static/
    ├── css/
    │   └── styles.css
    └── js/
        ├── api.js
        ├── app.js
        └── render.js
```

- `index.html`: タイトル画面とゲーム画面の意味構造
- `styles.css`: 配色、レイアウト、カード表現、レスポンシブ表示
- `api.js`: HTTP API、Bearerセッション、WebSocket接続
- `app.js`: 画面遷移、フォームイベント、セッション復元
- `render.js`: 公開ゲーム状態からDOMへの描画と操作選択肢の生成

ゲームルールと状態変更はブラウザ側へ移していない。
すべての操作はHTTP APIへ送られ、サーバーで再検証・実行される。
WebSocketは閲覧者ごとの更新通知にだけ使用する。

## セッション

入室時のトークンは `sessionStorage` に保存する。
同じタブを再読み込みした場合は対戦へ復帰できるが、
別タブへはセッションを自動共有しない。このため、同じブラウザの
別タブからプレイヤー1・プレイヤー2として動作確認できる。

サーバーを再起動するとバックエンドのメモリ状態とセッションが消えるため、
画面は無効な保存済みセッションを破棄してタイトルへ戻る。

## 対応操作

- 対人ルーム入室、CPU初級・中級・上級の作成
- 手札破棄、ドロー、能力使用、ターン終了、ESPER宣言
- 全能力の選択操作
- プリサイエンスの山札順序指定
- サイコキネシス対象の盤面クリック、選択強調、実行前確認
- 再戦、退出、チャット
- ボタンで必要なときだけ展開するバトルログ
- WebSocketによる対戦相手・CPU操作のリアルタイム反映
- 対戦相手の手札入れ替え・能力結果を中央へ順次表示
- 自分の手札・盤面が変更された場合の強調通知
- 先攻決定・通常のターン交代時の「あなたの番」「相手の番」通知
- イベントIDによるHTTP・WebSocket間の通知重複排除
- 再読み込み後のセッション復元

## 対戦中の中央通知

`render.js` は公開状態の `action_events` をID順のキューへ積み、
通常行動、能力、自分への影響を色分けして中央に表示する。
タイムリープも同じキューを使うため、複数の中央表示は重ならない。
通常のターン交代は能力結果の後に表示し、追加ターンでは専用演出だけを表示する。
通知は操作を妨げず、表示後にフェードアウトする。初回接続、再接続、
再読み込みでは過去イベントを再生せず、その後に届いたイベントだけを表示する。

## 秘匿情報

ブラウザは `PublicGameState` のみを描画する。
通常時の相手手札、除外カード、相手の裏向き捨て札などは、
カード名自体がレスポンスに含まれず `null` または枚数として届く。
JavaScriptはそれをカード裏面として表示する。

## 現段階の制約

- ブラウザ自動操作によるE2Eテストはまだ追加していない。
- バックエンド状態は単一プロセスのメモリに保存される。
