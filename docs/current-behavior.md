# ESPER 現行動作仕様

## 目的

この文書は、Flet 版から HTML/CSS/JavaScript 版へ移行する前の現行挙動を固定するための仕様書である。
移行中は、明示的に変更を合意した項目を除き、この文書に記載された挙動を維持する。

対象コード:

- `main.py`
- `game_logic.py`
- `ui_views.py`

## 1. 実行方式と状態管理

- Flet の Web アプリとして起動する。
- 待受ポートは環境変数 `PORT`、未指定時は `8000` を使用する。
- 対戦部屋はプロセス内の `GAME_ROOMS` 辞書で管理する。
- サーバーを再起動すると、すべての対戦部屋と対戦状態が消える。
- 複数プロセス間では対戦部屋が共有されない。
- 同じルーム ID の画面更新は Flet PubSub で通知する。

## 2. 入室とプレイヤー

- 名前の初期入力値は `プレイヤー`。
- ルーム ID は必須で、空の場合は入室できない。
- ルーム ID の空白除去、文字種変換、大文字小文字の統一は行わない。
- 存在しないルーム ID へ入室すると、新しいゲームを作成する。
- 最初の入室者が `p1`、2人目が `p2` になる。
- 3人目以降は満員として拒否する。
- 2人目が入室すると先攻抽選へ進む。
- CPU 戦では人間が `p1`、CPU が `p2` になる。
- CPU 難易度は `easy`、`normal`、`hard` の3種類。
- CPU ルーム ID は、開始時点の秒単位 Unix 時刻から生成する。

## 3. カード構成と初期状態

- カードは次の7種類。
  - クレヤボヤンス
  - タイムリープ
  - サイコキネシス
  - プリサイエンス
  - テレポート
  - ヒーリング
  - カモフラージュ
- 各種類8枚、合計56枚で山札を作り、シャッフルする。
- 山札の先頭から3枚をゲーム外カードとして除外する。
- 残った山札の末尾から、`p1`、`p2` の順に6枚ずつ配る。
- 配札後の山札は41枚。
- 初期ターンは `p1` だが、対人戦開始時に先攻抽選で上書きする。
- 2人目の入室までは `WAITING`。

## 4. 先攻抽選

- 2人揃うと `DECIDING_TURN` になる。
- `p1` の画面が1.5秒の待機処理を1度だけ開始する。
- `p1` と `p2` からランダムに先攻を決定する。
- 抽選後は先攻プレイヤーの `DISCARD` から開始する。

## 5. 通常ターン

通常ターンは次の順に進む。

1. `DISCARD`: 手札から1枚捨てる。
2. `DRAW`: 山札から手札が6枚になるまで補充する。
3. `THINK`: 能力を使うか、ターンを終了するか選ぶ。
4. 能力を使わない場合は相手の `DISCARD` へ移る。

捨て札の公開状態:

- 各プレイヤーについて、捨て札エリアにある裏向きカードが5枚未満なら、通常捨て札は裏向き。
- 捨て札エリアにある裏向きカードが5枚に達している間、通常捨て札は表向き。
- 能力のコストや能力によって捨てるカードは表向き。
- 複数枚を同時に捨てたカードは、1つの捨て札グループとして扱う。

補充:

- 手札が6枚未満の場合だけ山札の末尾から補充する。
- 手札が6枚以上の場合は補充しない。
- 山札が途中で空になった場合、存在する枚数だけ補充する。

## 6. 能力発動

### 通常の能力発動

- 同名カード2枚を表向きの1グループとして捨てる。
- カモフラージュそのものは、通常の同名2枚能力として発動できない。
- 山札が1枚以下の場合、ヒーリング以外の通常能力ボタンは無効になる。

### カモフラージュ

- カモフラージュ2枚と、カモフラージュ以外のカード1枚を表向きで捨てる。
- 3枚目に選んだカードの能力として処理する。
- 山札が2枚以下の場合、ヒーリング以外への擬態は無効になる。

### クレヤボヤンス

- 相手の手札と、相手の裏向き捨て札から最大2枚を選ぶ。
- 選択したカード名を発動者だけが確認する。
- 0枚を選択したまま完了することもできる。
- 確認後、発動者の手札を6枚まで補充してターンを終了する。

### タイムリープ

- 発動者の手札を6枚まで補充する。
- 現在のプレイヤーを変更せず、再びそのプレイヤーの `DISCARD` へ移る。
- 追加ターンでも能力を使用できる。

### サイコキネシス

- 相手の手札から1枚選び、表向きの捨て札にする。
- 続いて、相手の「1枚だけで構成された裏向き捨て札グループ」を1つ選び、相手の手札へ戻す。
- 戻せる裏向き捨て札がない場合、最初に捨てさせたカードを相手の手札へ戻し、表向き捨て札への追加を取り消す。
- 処理後、発動者の手札を6枚まで補充してターンを終了する。

### プリサイエンス

- 山札の上から最大3枚を閲覧する。
- 閲覧したカードについて、山札の上から1番目、2番目、3番目になる順番を指定する。
- 指定した順番で3枚を山札の上へ戻す。
- その後、発動者の手札を6枚まで補充してターンを終了する。
- 通常の2枚ペアで発動した場合、能力コストによって手札が4枚になっているため、指定した1番目と2番目を直後の補充で引く。
- 指定した3番目は山札の一番上に残り、次のドロー対象になる。

現行コードは通常の2枚ペア発動では、選んだ2枚を手札へ直接加え、残り1枚を山札へ戻すことで、結果的にほぼ同じ手札と山札を作っている。ただし「3枚を山札へ戻した後、通常の補充処理で引く」という処理順にはなっていない。第2段階で能力処理をサービス層へ抽出する際、上記の正規手順へ統一する。

### テレポート

- 7種類から1種類を宣言する。
- 相手の手札にある宣言カードをすべて取り除き、表向きの1グループとして捨てる。
- 相手と発動者の両方を6枚まで補充する。
- 両者の補充に必要な合計枚数が山札枚数を超える場合、その時点で引き分けにする。
- 宣言カードが相手に0枚でも能力は完了する。

### ヒーリング

- 両プレイヤーの捨て札から最大3枚を選ぶ。
- 0枚のまま完了することもできる。
- 相手の裏向きカードは、選択時もカード名を表示しない。
- 選んだカードを元の捨て札グループから取り除き、空になったグループは削除する。
- 選んだカードを山札へ加え、山札全体をシャッフルする。
- 発動者の手札を6枚まで補充してターンを終了する。

## 7. ESPER 判定

- 同種カード5枚以上で ESPER 達成。
- カモフラージュ5枚以上でも ESPER 達成。
- カモフラージュ2枚につき、他種類のカード1枚分として数える。
- 端数のカモフラージュ1枚はワイルドカードとして数えない。
- ゲーム終了前であれば、自分のターンでなくても宣言ボタンが表示される。
- 宣言すると即座に `GAME_CLEAR` となる。

## 8. ターン終了と自動決着

能力処理または能力を使わないターン終了時に、次の順で判定する。

1. 山札が0枚ならゲーム終了。
2. どちらかの捨て札グループが18組以上ならゲーム終了。
3. タイムリープ中なら同じプレイヤーの追加ターン。
4. それ以外は相手のターン。

自動決着時:

- まず双方の ESPER 達成を確認する。
- 両者達成なら引き分け。
- 片方だけ達成なら、そのプレイヤーの勝利。
- どちらも未達成なら、手札の同種枚数を降順にしたリストを辞書式比較する。
- 構成が完全に同じ場合は引き分け。

## 9. CPU

- CPU は自分の操作開始前に1秒待つ。
- ESPER 達成時は、操作より先に宣言する。
- 初級は基本的にランダムで、能力を使用せずターンを終える。
- 中級は使用可能な能力があれば70%の確率で能力選択へ進む。
- 上級は使用可能な能力があれば95%の確率で能力選択へ進む。
- 上級は通常捨て札、テレポート、ヒーリング、プリサイエンスで手札や公開情報を考慮する。
- 能力の対象選択を含め、CPU 処理完了後に両画面へ更新通知する。

## 10. ログとチャット

- ログには時刻、役割、プレイヤー名、アイコン、本文を保存する。
- ログ履歴は新しいものから表示する。
- チャットは空白だけの本文を送信できない。
- チャット履歴は対戦部屋のゲームオブジェクト内に保存する。
- チャット件数とログ件数に上限はない。

## 11. ゲーム終了、再戦、退出

- `GAME_CLEAR` または `GAME_OVER` になると、相手の手札と除外カードを公開する。
- CPU 戦ではCPU側の再戦承認を自動追加する。
- 両プレイヤーが再戦を承認するとゲームをリセットし、再度先攻抽選へ進む。
- 再戦時もプレイヤー名、CPU設定、チャット履歴、ログ履歴は保持する。
- 再戦時に山札、手札、捨て札、能力の一時選択、追加ターン、再戦承認は初期化する。
- 退出すると部屋を `ROOM_DISBANDED` にし、参加者へ通知した後、`GAME_ROOMS` から削除する。

## 12. 現行制約

- 状態はメモリ内だけに存在し、永続化しない。
- ルーム単位の排他制御はない。
- ブラウザ再読み込み後に、既存プレイヤーとして復帰する仕組みはない。
- 名前やチャット本文の長さ制限はない。
- ルーム ID に認証情報や署名はない。
- CPU 動作と先攻抽選にはバックグラウンドスレッドを使用する。

## 13. 回帰テスト範囲

`tests/test_game_logic.py` では、UIに依存しない次の挙動を固定する。

- カード構成、配札、カード総数保存
- 手札ソート
- ESPER 判定とカモフラージュ換算
- 手札補充
- プレイヤー、相手、捨て札の参照
- ログ追加
- 自動勝敗判定
- 通常ターンと追加ターンの切り替え
- 山札切れと捨て札上限
- 再戦時に初期化する状態と保持する状態

Flet のイベントハンドラー内にある能力処理は、第2段階でサービス層へ抽出した後、そのサービス層に対する回帰テストを追加する。
